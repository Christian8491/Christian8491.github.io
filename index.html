Local Barycentric Coordinates

A perspective with modern OpenGL

Abstract

Barycentric coordinates yield to powerful and simple paradigm to interpolate data values on polyhedral domains. But with Local Barycentric coordinates we produce an interpolation between control points and inner points locally and win computational cost and storage requirements, inner points represent interior points of the domain as an affine combination of a set of control points.

The present project calculates the deformation of models, particularly deformed boxes of 2D or 3D models. This deformation is carried out locally with respect to the whole domain of the model that is applied to the different mathematical methods so that the domain can be completely measured with respect to a single deformation at a control point. Separate the variation level by different colorations.

Usage

  • Press the button "Load 2D Cage" or "Load 3D Cage" to load a cage obj file from folder 'data'. The vertices from the obj files are used as cage vertices.
  • Set the target number of cells for triangulation, and press the button "Triangulate" to compute the triangulation.
  • Set the solver parameters if necessary, then press the button "Compute LBC" to compute the coordinates.
  • Afterwards, check the checkbox "Display Weights", and click a control point to view its coordinate values via color coding.

Detailed Report

Implementation details

For the implementation we have used the following:

  • Language C++ 11
  • Framework Qt 5.2.0 MinGW 32 bit
  • External Libraries:
    • Eigen: For Lineal Algebra Operations
    • Triangle: For Delaunay Triangulation in 2D
  • Some instances of cage-based in 2D

Limitations

One of the options to modify initially was for the part of parallelization, which included changing the OpenMP part to CUDA. This change was not made for the following reasons:

  • The graphic part is not related anywhere with parallel implementation.
  • OpenMP is used for calculations such as interpolations and iterative aspects, in which it works with an amount no greater than 110 control points and for that reason using CUDA will not provide benefits.
  • The Eigen library works internally with OpenMP, this means that if you want to use CUDA for the present project then you should modify much of the library to work with CUDA, which is a very complicated issue.

    • Achievements

    Below are the parts that could be achieved with respect to what was mentioned in the first presentation:

    • The modern OpenGL part (using shaders) was implemented, leaving aside the previous implementation with old OpenGL.
    • The previous implementation only loaded models in 2d, that is, if we wanted to load models in 3D there was no support, that is why a class was implemented for the reading in particular of 3D geometric models.
    • Additional implementation to triangulate the model of the box on its surface.

    Future Work

    As a first future work, we would finish the part of the calculation algorithm for 3D models, since we only reached triangulation on the surface, but not load the texture once the calculations were processed.


    Another very interesting contribution would be the implementation of paper Interactive Cage Generation for Mesh Deformation which generates a box deformation for any three-dimensional model, this would be a great contribution because for the present work we only have a box model (for the armadillo) and then apply the algorithm of calculations of LBC to see the level of deformation with respect to each control point.

    Some math usage

    We will show some concepts necessary for the understanding of the topic.

    • Affine Combinations.
    • Convex minimum optimization.
    • High order Laplacian.
    • Properties of coordinate functions (to good interpolation)
      • Reproduction
      • Partition of unity
      • Non-negativity
      • Lagrange property
      • Linearity
      • Smoothness
      • Locality
    • Voronoi diagrams.
    • Delaunay Triangulation.
    • Isoperimetric inequality.
    • Measure of Sparsity with l0-norm and lp-norm.
